Open Ticket Agency is an open source code base for smart contracts and reference web applications that implement ticketing systems on the Ethereum blockchain.

Goal

The goal of this project is to create a reference implementation of a ticket sales system on the Ethereum blockchain. This system will be designed to be as flexible as possible so that it can be used as is or as a base for other projects. As such, it can be used by any event organizer to build their own ticket sales system or to customize it to their needs.

Features

Open Source

It's open source. Anyone can use and modify the code for their own needs.

Decentralized

It's a decentralized. Service is non-stop, data is stored on the blockchain.

Web App

It's based on craftkit, a new library for readability and maintainability.

Structure

Smart Contract

Open Ticket Agency consists of several smart contracts. Factory contract is a contract that creates agency contract. This is usually deployed by the owner of the system or your self. Agency contract manages show contracts. Show contract manages ticket sales.

Deployer App

Deployer app is a web application that deploys agency contract for your business.

Agency App

Agency app is a web application that creates and manages show contracts.

Show Manager App

Show manager app is a web application that manages ticket sales.

User App

User app consists of two web applications. One is for buying tickets used by the general public, and the other is for checking tickets used by the event staff.

Development Cycle

Open Ticket Agency is developed in an agile way. It is developed in a way that allows you to easily add new features and fix bugs.

Roadmap

First Cycle

Now we are developing the first cycle. The first cycle aims to provide a basic ticketing system until the end of 2022.

  • Smart Contract
  • Deployer App
  • Agency App
  • Show Manager App
  • User App

last update: 2022-10-29

Second Cycle

The second cycle aims to provide a ticketing system that can be used by a large number of people. Smart contract will support almost all features of the ticketing system. The web application will be limited to the basic features in the first cycle. So we would focus on improving the web application in the second cycle.

Third Cycle

The third cycle aims to modularize contract and web application. In this cycle, we may focus on abstracting the web application stucture.

Development goes on

We will continue to develop the system. We will add new features and fix bugs.
